# UMI/Duplex Tool Landscape

## UMI-tools
- **Language:** Python
- **Approach:** Network-based methods (directional, adjacency, cluster) on aligned BAM files grouped by genomic coordinate
- **Strengths:** Most cited academic tool
- **Weaknesses:** Slow, memory-hungry, no duplex support
- **Method:** Build graphs where nodes are UMIs and edges connect UMIs with edit distance ≤ threshold, then define read groups from the network

## fgbio
- **Language:** Scala/JVM
- **Approach:** `GroupReadsByUmi` → `CallDuplexConsensusReads` pipeline
- **Strengths:** De facto standard for duplex UMI consensus in clinical settings. 300k+ Bioconda downloads. Supports single-strand and duplex, flexible barcoding schemes.
- **Weaknesses:** JVM startup/memory overhead. Requires alignment first. Limited configurability around duplex consensus calling parameters. Limited per-family metadata output.
- **Key parameter:** `min-reads` in `CallDuplexConsensusReads` — e.g., `2 1 1` requires 2 reads total with ≥1 forward and ≥1 reverse for true duplex consensus. `1 1 0` allows simplex consensus (riskier for rare variants).
- **Note:** Consensus base quality information generated by fgbio was not used in variant calling in benchmarks — suggests room for improvement.

## UMICollapse
- **Language:** Java
- **Approach:** N-grams and BK-trees for fast UMI dedup
- **Strengths:** Fastest current tool. Orders-of-magnitude faster than UMI-tools (hours/days → minutes).
- **Weaknesses:** Works on aligned BAM only. No duplex UMI strand-pairing logic.

## Sentieon ctDNA Pipeline
- **Language:** C (commercial, closed-source)
- **Strengths:** ~4× faster than Picard for consensus generation. Overall FASTQ→VCF pipeline 2–3× faster than alternatives.
- **Weaknesses:** Proprietary and expensive.

## Jellyfish
- **Language:** C++
- **Approach:** Lock-free hash table k-mer counting with compare-and-swap CPU instructions
- **Strengths:** Extremely fast and memory-efficient. Order of magnitude better than other k-mer counters.
- **Weaknesses for duplex ctDNA work:**
  - Canonical k-mer collapsing (`-C` flag) erases strand information
  - No quality-weighted counting (Q30 and Q10 k-mers look identical)
  - No read-of-origin tracking (can't tell 5 families from 5 PCR duplicates)
  - Binary `.jf` format is opaque

## km (find_mutation)
- **Language:** C++
- **Approach:** Graph walking over k-mer space using Jellyfish counts against target sequences
- **Originally designed for:** RNA-seq investigation at IRIC
- **Strengths:** Clever approach — asks "what sequence paths are supported by k-mer evidence?" Handles indels cleanly, no alignment bias in repeats.
- **Weaknesses:** Single-threaded. `--ratio` threshold is a global cutoff with no statistical model. Calibrated for RNA-seq, not ultra-deep ctDNA panels.

## kmtools
- **Purpose:** Parallelization wrapper for km. Chunks target sequence list, runs parallel km processes, merges results.
- **Additional:** `filter` command matches output against known variant reference. `plot` gives VAF histograms, type distributions, per-sample summaries.

## References

Smith, T., Heger, A. and Sudbery, I. (2017) 'UMI-tools: modeling sequencing errors in Unique Molecular Identifiers to improve quantification accuracy', Genome Research, 27(3), pp. 491–499. doi: 10.1101/gr.209601.116. [Primary reference for UMI-tools; source of network-based methods (directional, adjacency, cluster), Python implementation, and alignment-based grouping approach.]

Fulcrum Genomics (2024) fgbio toolkit. GitHub repository. Available at: https://github.com/fulcrumgenomics/fgbio (Accessed: March 2026). [Primary reference for fgbio; source of duplex consensus pipeline, `GroupReadsByUmi` / `CallDuplexConsensusReads` design, `min-reads` parameter, Bioconda download statistics, and JVM overhead characteristics.]

Liu, D. (2019) 'Algorithms for efficiently collapsing reads with Unique Molecular Identifiers', PeerJ, 7, p. e8275. doi: 10.7717/peerj.8275. [Primary reference for UMICollapse; source of N-gram and BK-tree deduplication approach and the orders-of-magnitude speed advantage over UMI-tools.]

Freed, D., Aldana, R., Weber, J.A. and Edwards, J.S. (2017) 'The Sentieon Genomics Tools: a fast and accurate solution to variant calling from next-generation sequence data', bioRxiv. doi: 10.1101/115717. [Primary reference for Sentieon; source of the ~4× speed improvement over Picard and overall 2–3× pipeline speedup figures.]

Marçais, G. and Kingsford, C. (2011) 'A fast, lock-free approach for efficient parallel counting of occurrences of k-mers', Bioinformatics, 27(6), pp. 764–770. doi: 10.1093/bioinformatics/btr011. [Primary reference for Jellyfish; source of lock-free hash table with compare-and-swap design, canonical k-mer `-C` flag behaviour, and binary `.jf` format characteristics.]

Bourgey, M. et al. (2019) 'km: find-mutation', GitHub repository. Available at: https://github.com/iric-soft/km (Accessed: March 2026). [Primary reference for km; source of the graph-walking k-mer variant detection approach, Jellyfish dependency, `--ratio` threshold design, and RNA-seq/IRIC origin.]
