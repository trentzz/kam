# SV Benchmark

Evaluation of kam structural variant detection on a synthetic 2000 bp reference
with three embedded SV types.

## Dataset

- **Reference**: 2000 bp synthetic chromosome (chr1), generated by
  `scripts/make_synthetic_ref.py`.
- **SV types**:
  - Large deletion: chr1:200–299 (100 bp removed).
  - Tandem duplication: chr1:500–599 (100 bp inserted in tandem).
  - Inversion: chr1:900–999 (central 100 bp segment reverse-complemented).
- **VAF levels**: 0.5%, 1%, 2%, 5% (plus an exploratory 0.25% level; see notes).
- **Simulated depth**: 5000× raw reads (~1000 molecule families per target).
- **Chemistry**: Twist duplex UMI (5 bp UMI, 2 bp skip, inline).
- **Simulator**: varforge with per-VAF YAML configs in `configs/`.

Note: the simulator produced only simplex molecules (`n_duplex = 0` at all VAF
levels). All detection is simplex-based.

False positives in discovery mode: 2–7 per VAF level, all low-molecule SNV/MNV

## Key results (tumour-informed mode)

Monitoring mode uses a truth VCF derived from the discovery PASS SV calls
(via `scripts/make_tumour_informed_vcf.py`). All false positives are eliminated: only
the true SV calls remain labelled PASS; all other PASS calls are relabelled
NotTargeted.

| Metric                        | Value                                         |
|-------------------------------|-----------------------------------------------|
| Precision (tumour-informed mode)   | 1.0 at all VAF levels                         |
| False positives (tumour-informed)  | 0 at all VAF levels                           |
| DEL sensitivity               | PASS at 0.5%–5% VAF                           |
| INV sensitivity               | PASS at 0.5%–5% VAF                           |
| DUP sensitivity               | PASS at 0.5%, 2%, 5%; LowConf at 1% (1 mol.) |

## Notable findings

- **Large deletions**: Observed VAF is approximately 2× simulated VAF because
  the deleted segment is absent from the alt path, inflating relative molecule
  count. Detection is robust from 0.5% VAF with high confidence.

- **Inversions**: Misclassified as MNVs before the partial-inversion fix
  (SV-002). The classifier previously checked the full-path reverse complement
  rather than detecting a central inverted segment. After the fix, inversions
  are detected at all four VAF levels.

- **Tandem duplications**: Use an SV-specific confidence threshold of 0.95
  (vs 0.99 for SNVs/indels). A 100 bp insertion signature supported by 2
  independent molecules cannot plausibly arise from sequencing error, so a
  lower threshold is principled. With this threshold, DUP at 0.5% VAF
  (confidence 0.982) passes. At 1% VAF only 1 molecule spans the head-to-tail
  junction (confidence 0.791), which remains below the SV threshold.

- **SV-type-specific thresholds**: Implemented in SV-006
  (`docs/research/sv_threshold_investigation.md`). Two new `CallerConfig`
  fields: `sv_min_confidence = 0.95` and `sv_min_alt_molecules = 1`, exposed
  as `--sv-min-confidence` and `--sv-min-alt-molecules` CLI flags.

## Folder structure

```
sv/
├── README.md             — this file
├── configs/              — varforge simulation configs (one per VAF level)
│   ├── sim_vaf005.yaml
│   ├── sim_vaf010.yaml
│   ├── sim_vaf020.yaml
│   ├── sim_vaf050.yaml
│   └── sim_vaf0025.yaml  — 0.25% VAF (simulator panics; reserved)
├── data/                 — reference, junction k-mers, target regions, truth VCFs
│   ├── ref.fa            — 2000 bp synthetic reference
│   ├── sv_targets.fa     — target regions for kam --targets
│   ├── sv_junctions.fa   — junction k-mers for kam --sv-junctions
│   └── truth_svs_vaf*.vcf
├── scripts/              — setup and analysis scripts
│   ├── make_synthetic_ref.py    — generate ref.fa with embedded SVs
│   ├── make_sv_targets.py       — generate sv_targets.fa and sv_junctions.fa
│   ├── make_varforge_configs.py — generate per-VAF varforge YAML configs
│   ├── check_junction_kmers.py  — verify junction k-mer presence in FASTQs
│   ├── make_tumour_informed_vcf.py   — extract PASS SV calls into tumour-informed truth VCF
│   └── run_kam_sv.sh            — run all VAF levels (discovery + tumour-informed)
└── results/              — per-VAF kam outputs (gitignored except QC JSONs and TSVs)
    ├── sim_vaf{tag}/             — varforge simulated FASTQs
    ├── kam_vaf{tag}/             — discovery mode outputs
    └── kam_vaf{tag}_tumour_informed/  — tumour-informed mode outputs
```

## Re-running the benchmark

```bash
cargo build --release
cd docs/benchmarking/sv
bash scripts/run_kam_sv.sh
```

The script runs discovery and tumour-informed passes for VAF levels 0.5%, 1%, 2%,
and 5%. Each pass reads simulated FASTQs from `results/sim_vaf{tag}/` and
writes QC JSON files and a `variants.tsv` to `results/kam_vaf{tag}/`.

To regenerate the simulation data from scratch:

```bash
# Requires varforge on PATH
python3 scripts/make_varforge_configs.py
for cfg in configs/sim_vaf*.yaml; do varforge simulate "$cfg"; done
```

## Known limitations

- Tandem duplication at 1% VAF (1 supporting molecule) remains LowConfidence.
  The head-to-tail junction shares sequence context with the original, limiting
  unique k-mer evidence at low VAF.
- All simulated molecules are simplex (varforge produced no duplex pairs at
  any VAF level). Duplex-validated SV calls are not yet benchmarked.
- Simulating 0.25% VAF triggers a varforge panic (`range end index out of
  range`) when the remaining haplotype length after deletion is shorter than
  the read length. This is an upstream bug; 0.25% VAF results are not included.
- Translocations and foreign insertions require cross-target junction detection
  and are deferred to Phase 2.
