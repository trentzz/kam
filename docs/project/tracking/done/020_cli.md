# Task 020: CLI with Clap

## Location
`kam/src/main.rs` and `kam/src/cli.rs`

## What to implement

The main CLI using clap with subcommands matching the pipeline stages. Each subcommand takes appropriate inputs and calls the corresponding crate.

## Interface

```
kam assemble   --r1 <R1.fq> --r2 <R2.fq> --output <molecules.bin> [options]
kam index      --input <molecules.bin> --targets <targets.fa> --output <index.bin> [options]
kam pathfind   --index <index.bin> --targets <targets.fa> --output <paths.bin> [options]
kam call       --paths <paths.bin> --output <variants.tsv> [options]
kam run        --r1 <R1.fq> --r2 <R2.fq> --targets <targets.fa> --output-dir <dir> [options]
```

`run` is the full pipeline in one command (calls all stages sequentially with zero-copy data passing).

## Subcommand options

### assemble
- `--r1`, `--r2` ‚Äî input FASTQ files (required)
- `--output` ‚Äî output file (required)
- `--chemistry` ‚Äî preset: `twist-umi-duplex` (default)
- `--min-umi-quality` ‚Äî Phred threshold for UMI bases (default: 20)
- `--min-family-size` ‚Äî minimum reads per family (default: 1)
- `--min-template-length` ‚Äî optional minimum template length
- `--log-dir` ‚Äî log output directory
- `--log` ‚Äî enable specific logs (repeatable)
- `--threads` ‚Äî number of threads

### index
- `--input` ‚Äî consensus molecules (from assemble)
- `--targets` ‚Äî target sequences FASTA
- `--output` ‚Äî k-mer index file
- `-k` ‚Äî k-mer size (default: 31)

### pathfind
- `--index` ‚Äî k-mer index file
- `--targets` ‚Äî target sequences FASTA
- `--output` ‚Äî scored paths file

### call
- `--paths` ‚Äî scored paths file
- `--output` ‚Äî variant calls file
- `--output-format` ‚Äî tsv,csv,json,vcf (comma-separated for multiple)
- `--min-confidence` ‚Äî minimum posterior probability
- `--strand-bias-threshold` ‚Äî Fisher's exact p-value cutoff

### run
- Combines all of the above
- `--output-dir` ‚Äî directory for all outputs
- `--qc-output` ‚Äî QC JSON file

## Implementation

Use clap derive API:

```rust
use clap::{Parser, Subcommand};

#[derive(Parser)]
#[command(name = "kam", about = "Alignment-free variant detection for duplex UMI sequencing")]
struct Cli {
    #[command(subcommand)]
    command: Commands,
}

#[derive(Subcommand)]
enum Commands {
    Assemble { ... },
    Index { ... },
    Pathfind { ... },
    Call { ... },
    Run { ... },
}
```

For now, each subcommand can print "not yet implemented" for the actual logic ‚Äî the CLI structure and argument parsing is the deliverable.
