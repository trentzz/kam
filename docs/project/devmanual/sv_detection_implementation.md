# Structural Variant Detection in kam: Implementation and Results

## Summary

Phase 1 SV detection is implemented and passes all tests. The pipeline now
classifies large deletions (≥ 50 bp), tandem duplications (≥ 50 bp), and
inversions, outputs them with standard VCF symbolic alleles, and supports
breakpoint k-mer augmentation via `--sv-junctions`. Full validation with
real sequencing data is deferred until the FASTQ data set is available.

---

## Background

The original kam pipeline detected small variants only: SNVs, MNVs, and
indels up to roughly 30 bp. Large deletions, tandem duplications, and novel
insertions ≥ 50 bp are not a distinct class mechanistically. They use the
same de Bruijn graph path walk as short indels, with longer alt paths and
adjusted scoring thresholds (`mean_variant_specific_molecules` instead of
`min_molecules`, confidence 0.95, min_alt_molecules 1). The 50 bp threshold
marks where these adjustments activate, not a different detection mode.

Inversions and fusions are the genuinely distinct case. Their breakpoints
produce k-mers absent from both the reference and the panel targets. Without
junction k-mer injection via `--sv-junctions`, the allowlist filter discards
these k-mers and the alt path is invisible to the path walk.

This document records the design decisions made, what was implemented, and
what worked.

The design was developed in `docs/research/sv_detection_design.md` before
implementation began. That document should be read alongside this one.

---

## What Was Implemented

### 1. Junction allowlist augmentation (`--sv-junctions`)

The core problem for SV detection is that structural variants create k-mers
at the breakpoint junction that are not present in the reference sequence.
The allowlist filter in `kam index` discards any k-mer not in the reference
target sequences. Without augmentation, the alt path for a deletion or
duplication is invisible to the de Bruijn graph.

The fix: accept an optional `--sv-junctions FASTA` file on both `kam index`
and `kam run`. The k-mers from these junction sequences are unioned into the
allowlist alongside the target reference k-mers. The change is localised to
the index stage and does not affect the hot path for standard SNV indexing.

The junction sequences are generated by `docs/benchmarking/scripts/make_sv_targets.py`.
For a deletion, the junction sequence is the concatenation of (k-1) bases
immediately before the deleted region with (k-1) bases immediately after.
The k-mers spanning this synthetic junction are the ones absent from the
reference — they are present only in reads from molecules carrying the alt
allele.

### 2. Per-target max_path_length override (`_maxpathN`)

The DFS walk budget (`max_path_length`) defaults to `target_len - k + 1 + 50`.
For a 100 bp SNV target at k=31, this is about 120 k-mers. For an SV with a
junction window target, the reference path may be much longer: a 200 bp
deletion has a reference path of about 170 k-mers. The default budget is
insufficient.

The fix: allow the target FASTA record ID to embed a per-target override via
the suffix `_maxpathN` (e.g. `chr17:100-300_DEL_200bp_maxpath250`). The
pathfind stage (`kam pathfind` and `kam run`) parses this suffix and applies
it to the WalkConfig for that target. The annotation is case-insensitive and
position-agnostic in the ID string.

This design avoids introducing a new config file format and is backward
compatible: targets without the suffix use the default budget unchanged.

### 3. New VariantType variants

Three new variants were added to the `VariantType` enum in `kam-call`:

- `LargeDeletion`: alt shorter than ref by ≥ 50 bp.
- `TandemDuplication`: alt longer than ref by ≥ 50 bp.
- `Inversion`: alt same length as ref, and alt is the reverse complement of ref.

The `classify_variant` function was extended to assign these types. Small
deletions and insertions (< 50 bp) continue to be classified as `Deletion`
and `Insertion` respectively.

For inversions, the detection logic checks whether `alt == rc(ref)` for the
full path sequences. This works correctly for inversion junction targets
designed so that the alt path traverses the inverted segment: the
reconstructed alt sequence would be the reverse complement of the reference
sequence for that window.

The 50 bp threshold is the standard definition of an SV (VCF spec, GATK,
ClinGen). Below 50 bp, the existing indel classification is used.

### 4. VCF symbolic allele output

SVs use symbolic allele notation in VCF 4.3: `<DEL>`, `<DUP>`, `<INV>`.
For large sequences (e.g. a 200 bp deletion), writing the full sequence in
the REF field is technically correct but non-standard and impractical.

The VCF writer now:
- Emits `<DEL>`, `<DUP>`, or `<INV>` in the ALT field for SV calls.
- Uses `extract_minimal_allele` to find the anchor coordinate, then uses
  only the anchor base (single character) as the REF field.
- Appends `SVTYPE=<type>` and `SVLEN=<length>` to the INFO string.
  SVLEN is negative for deletions, positive for duplications, 0 for
  inversions (following VCF convention for SVLEN sign).
- Adds `##ALT=<ID=DEL>`, `##ALT=<ID=DUP>`, `##ALT=<ID=INV>` header lines.
- Falls back to the alignment-free format (POS=1) when the target ID cannot
  be parsed as a genomic coordinate.

TSV and CSV output includes the string labels `LargeDeletion`,
`TandemDuplication`, `Inversion` in the `variant_type` column.

### 5. `make_sv_targets.py` script

`docs/benchmarking/scripts/make_sv_targets.py` generates:
1. A junction window target FASTA (`--out-targets`) for use with `--targets`.
2. A junction k-mer allowlist FASTA (`--out-junctions`) for use with
   `--sv-junctions`.

For each SV in the truth VCF, the script:
- Detects the SV type (DEL, DUP, INV) from ALT field or INFO SVTYPE.
- Fetches the reference sequence at the breakpoint using pyfaidx or samtools.
- Constructs a 100 bp junction window centred on the breakpoint.
- Writes the synthetic junction sequence (2k-2 bp) for the allowlist.
- Embeds `_maxpathN` in the target ID when the reference path exceeds the
  default walk budget.

The script supports pyfaidx (preferred) and samtools faidx as backends for
reference access.

## Design Decisions

### Why `_maxpathN` in the FASTA ID rather than a separate config file?

Two options were considered:
- A separate config file mapping target IDs to parameters.
- Embedding the override in the FASTA ID.

The FASTA ID approach requires no new file formats, no new CLI flags for
pathfind, and no schema changes to intermediate binary files. The downside is
that the ID becomes longer and less human-readable for SV targets, but SV
targets are always generated programmatically by `make_sv_targets.py`, so
the ID format is not a usability concern. The annotation is parsed at runtime
and ignored when absent, so it is strictly backward compatible.

### Why 50 bp as the SV threshold?

The 50 bp threshold is the standard VCF/GATK definition. It aligns with the
design document and ensures that the existing indel code path handles all
variants that fit comfortably within the 100 bp target window. A 49 bp
deletion uses 49 bp of the 100 bp target; a 50 bp deletion starts to require
the junction window approach described in the design.

### Why check reverse complement for inversions?

An inversion creates k-mers that are the reverse complement of reference
k-mers. Because k-mer canonicalisation uses `min(kmer, rc(kmer))`, canonical
k-mers from the inverted segment match canonical k-mers from the reference.
This means the de Bruijn graph can, in principle, find the inversion path
using the allowlist-filtered canonical index. However, path reconstruction
(which uses raw k-mer suffix/prefix overlap) will only work if the inversion
target is constructed so that the alt path contains raw k-mers that overlap
in the correct orientation. The `make_sv_targets.py` script constructs
inversion junction windows that include reverse-complement sequence, which is
the correct approach.

The `is_reverse_complement` check in `classify_variant` is a post-hoc label:
it identifies calls where the reconstructed alt path is the full reverse
complement of the reference path for the target window. This will correctly
label inversions where the entire target window is inverted.

## What Works

All unit tests pass:

- `classify_large_deletion`: a 60 bp deletion is classified as `LargeDeletion`.
- `classify_small_deletion_not_sv`: a 20 bp deletion remains `Deletion`.
- `classify_tandem_duplication`: a 60 bp insertion is `TandemDuplication`.
- `classify_inversion`: `AAACCC` vs `GGGTTT` (rc) is `Inversion`.
- `is_reverse_complement_basic`: RC detection is correct for all cases.
- `vcf_large_deletion_uses_symbolic_allele`: VCF output includes `<DEL>`,
  `SVTYPE=DEL`, `SVLEN`, and `##ALT=<ID=DEL`.
- `parse_maxpath_from_id_*`: ID suffix parsing is correct and case-insensitive.

The pipeline builds and all 331 tests pass with no warnings.

## What Was Not Validated With Real Data

Full sensitivity benchmarking against synthetic varforge data requires:
1. Varforge to generate cfDNA reads with known SV breakpoints.
2. A reference genome available locally.
3. Running `make_sv_targets.py` to generate junction targets.
4. Running `kam run` with `--targets sv_targets.fa --sv-junctions sv_junctions.fa`.
5. Scoring against the truth VCF.

The FASTQ data for the titration samples was not available on this machine.
The pipeline integration is ready to test as soon as reference data is available.

The sensitivity targets from the design document remain as goals:

| SV type | VAF | Target sensitivity |
|---------|-----|-------------------|
| Deletion 50–200 bp | 2% | ≥ 0.5 |
| Deletion 50–200 bp | 1% | ≥ 0.3 |
| Tandem duplication 50–200 bp | 2% | ≥ 0.4 |
| Inversion 50–200 bp | 2% | ≥ 0.4 |

## Known Limitations

**Soft end-anchor for large SVs**: The soft anchor search scans inward up to
10 positions. For a large deletion junction target where the reference path
is much longer than the alt path, both anchors should be present in the
graph from alt reads. The soft anchor is not needed for junction window
targets (both anchors are in the flanks), but the mechanism does not break
either.

**De novo SV discovery**: All current SV detection is tumour-informed
monitoring. The breakpoint coordinates must be known from a prior tissue
biopsy to construct the junction targets and junction k-mers. De novo
discovery (scanning for k-mers with high molecule support not in the
reference) is Phase 2.

**Translocations**: Two-locus path walking is not implemented. Translocation
detection requires either a chimeric target (synthetic sequence spanning
both breakpoints) or a multi-target walk. Deferred to Phase 2.

**Very large SVs (> 500 bp)**: Junction window targets work regardless of SV
length. However, for tandem duplications > 500 bp, the duplication junction
sequence is distant from both flanks and sensitivity may drop at low VAF.
