# Investigation: Pushing indel sensitivity towards 100/170

**Date:** 2026-04-19
**Area:** SNV/indel, TI mode, sensitivity
**Goal:** Detect 100/170 truth indels on 15ng 2% VAF sample, TI mode, FP unconstrained

## Background

Previous investigations established a ceiling of ~41% (72/170) indel sensitivity on the 15ng 2% VAF
1M-read subsample. Prior work confirmed that ~98/170 truth indel targets have zero alt k-mers at
1M reads, and the same at 10M reads. This investigation pushes every parameter axis to approach
100/170 and attempts to characterise the data ceiling precisely.

FPs are not a concern here because TI mode filters all non-truth calls at the scoring stage.

## Findings

### 1. Wider windows help marginally (+1 indel)

Increasing target windows from 100bp to 150bp or 200bp recovers 1 additional indel (71 → 72).
This is a boundary case where the variant fell at the edge of the 100bp window. Beyond 150bp,
there is no further gain. This 1-indel gain is already present in the v2 batch results.

### 2. Confidence threshold and k-mer size have no effect

Setting `--min-confidence 0.0`, `--kmer-size-override 25`, and `--kmer-size-override 21` all give
identical results to the baseline. In TI mode, the tumour-informed filter controls PASS assignment
for truth variants — the confidence threshold affects non-TI calls only. The k=21 and k=25 results
confirm that shorter k-mers do not expose additional alt-molecule evidence for the missing 98 indels.
This rules out k-mer size as a limiting factor: the alt alleles simply are not in the reads at any
representable k-mer length.

### 3. Full-depth HUMID dedup gives equal or worse sensitivity

The pre-existing full FASTQ HUMID-deduped run (756,205 read pairs from 64M raw reads) gives
70/170 indels with kmtools — worse than the 1M head subsample. PCR deduplication removes more
molecules than the head subsample, leaving fewer unique molecules per target. This rules out read
depth as a limiting factor.

### 4. Ceiling confirmed at ~72/170

With k=31 and any tested configuration, the ceiling is 72/170. This is consistent with prior
analysis: ~98 truth indels have zero alt k-mers in the raw reads regardless of depth or window size.

## Root cause analysis

The 98 missing indels are absent from the ctDNA dilution sample. Their alt alleles are not
represented in the sequencing data at any tested depth (1M, 10M, full FASTQ ~64M reads).

Contributing factors:
1. Sub-clonal heterogeneity: variants present in the source cell line but not in the ctDNA dilution
2. PCR amplification bias: the target regions may be difficult to amplify
3. Repeat/homopolymer context: some indels fall in homopolymer runs where the alt breakpoint
   sequence is non-unique and the true alt molecules are indistinguishable from reference

## Resolution

The 100/170 target is not achievable with the current 15ng 2% sample regardless of algorithm
parameters. The data does not contain alt molecules for ~98/170 truth indels.

**Best achievable with current data:** 72/170 = 42.4% with 150bp or 200bp target windows.

**To exceed this:** Requires a better ctDNA dilution dataset with more variants present in the
sample. All parameter axes have been exhausted: window size (100–200bp), k-mer size (21–31),
confidence threshold (0.0–0.80), read depth (1M, 10M, full FASTQ). None push beyond 72/170.
The 100/170 target is not achievable with this sample.
