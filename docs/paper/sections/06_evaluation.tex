\section{Evaluation}
\label{sec:evaluation}

Two implementation bugs were identified by code review and subsequently fixed.
We describe each and their impact on sensitivity.

% ── Strand bias overcounting ──────────────────────────────────────────────────
\subsection{Strand bias overcounting}
\label{sec:eval:strandbias}

The original implementation computed strand bias counts as the sum of
\texttt{n\_simplex\_fwd} and \texttt{n\_simplex\_rev} across all k-mers in
the alt path.
For a 100\,bp target with $k = 31$, the alt path contains approximately 70
k-mers.
A real variant supported by 5 forward-strand molecules and 0 reverse-strand
molecules therefore appears as 350 forward reads vs.\ 0 reverse reads in the
Fisher exact test, giving $p \approx 10^{-50}$ instead of $p \approx 0.5$.

This inflated sample size made the strand bias filter approximately 70$\times$
more sensitive than intended.
Any low-VAF variant where all supporting molecules happened to come from one
strand --- common at 2 alt molecules --- was filtered regardless of the true
per-molecule strand balance.

The fix is to use the per-path minimum rather than the sum: the minimum of
\texttt{n\_simplex\_fwd} across all k-mers approximates the number of
forward-strand molecules covering the full path.
This gives a calibrated per-molecule count as input to the Fisher test.

% ── Duplex orientation comparison ─────────────────────────────────────────────
\subsection{Duplex orientation comparison}
\label{sec:eval:duplex}

For a genuine duplex pair, the forward and reverse reads carry fingerprints
related by a 32-bit rotation:

\begin{equation*}
  \text{fp}_\text{rev} = \text{rotate\_left}(\text{fp}_\text{fwd},\; 32)
\end{equation*}

The original \texttt{fingerprints\_compatible} function computes
$(fp_1 \oplus fp_2).\text{popcount()} \le 4$.
For a genuine duplex pair, this XOR has approximately 32 bits set (the upper
and lower 32-bit halves encode different sequence regions), so the test always
returns false.
Forward and reverse reads from the same molecule were split into separate
fingerprint groups and never combined into a duplex molecule.

The fix adds a rotation check:
\texttt{fingerprints\_duplex\_compatible} returns true if either the direct
comparison or the 32-bit-rotated comparison passes the threshold.
This correctly groups genuine duplex pairs while keeping the collision detection
logic unchanged for same-strand reads.

At shallower depths (200K--500K reads) the duplex rate was near zero because
duplex pairs require both strands of the same fragment to be sampled.
Duplex yield grows with depth until fragment saturation.

% ── Sensitivity at sub-0.1\% VAF ─────────────────────────────────────────────
\subsection{Sensitivity at sub-0.1\% VAF}
\label{sec:eval:sensitivity}

Sensitivity is near zero at 0.001\% and 0.01\% VAF across all depths and
concentrations. At 0.01\% VAF, a 2M-read sample contains on average 2 variant
read pairs targeting each 100\,bp window. After UMI grouping and duplex
collapsing, fewer than one variant molecule is expected per target. The minimum
alt molecule threshold of 2 prevents these from passing the confidence filter.

Detecting at sub-0.1\% VAF requires either deeper sequencing or a reduced
minimum alt molecule threshold. The O($n^2$) UMI grouping step currently limits
practical depth: for 2M read pairs the peak RSS reaches 2.4\,GB; scaling
further would require switching to a hash-partition strategy (group reads by
UMI prefix, then compare within partitions), which would reduce expected
complexity to O($n$) and allow tens of millions of read pairs within a 4\,GB
budget.

% ── Overall assessment ────────────────────────────────────────────────────────
\subsection{Overall assessment}

With both implementation fixes applied, sensitivity extends to the 0.25--1\%
VAF range at 2M reads.
Indel sensitivity remains lower than SNV sensitivity at all VAF levels.
This is a structural characteristic of anchor-based graph walking: when a
variant path is a different length from the reference path, the end anchor
k-mer shifts position relative to the reference.
Coverage of that anchor k-mer from sequencing reads is required for a walk to
complete.
At 0.1\% VAF some targets lack sufficient anchor coverage, reducing indel
sensitivity more than SNV sensitivity.
Increasing depth further would partially close this gap.
